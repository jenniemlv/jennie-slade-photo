  <action>
Create `src/components/layout/Header.tsx` as a `'use client'` component. This is a single file containing both the desktop nav and the inline mobile menu overlay.

**Imports:**
- `'use client'` at the very top
- `useState`, `useEffect` from `'react'`
- `usePathname` from `'next/navigation'`
- `Link` from `'next/link'`
- `Menu`, `X`, `Instagram` from `'lucide-react'`

**Navigation data** — define as a const array at the top of the file:
```tsx
const NAV_LINKS = [
  { href: '/about', label: 'About' },
  { href: '/portfolio', label: 'Portfolio' },
  { href: '/contact', label: 'Contact' },
  { href: '/blog', label: 'Blog' },
] as const
```
Per D-02: exactly 4 links. No "Home" link. Clicking "JENNIE SLADE" navigates to `/`.

**State variables:**
- `scrolled` (boolean, default false) — toggles background from transparent to solid at 80px
- `hidden` (boolean, default false) — toggles header visibility on scroll direction
- `menuOpen` (boolean, default false) — mobile menu overlay

**Scroll effect** (`useEffect` with empty dependency array):
- Track `lastScrollY` via closure variable (not state)
- Add scroll listener with `{ passive: true }`
- On scroll:
  - Set `scrolled` to `currentScrollY > 80` (per UI-SPEC threshold)
  - If `delta > 10` AND `currentScrollY > 80`: set `hidden(true)` (hide on scroll-down)
  - If `delta < -5`: set `hidden(false)` (show on scroll-up)
  - Update `lastScrollY = currentScrollY`
- Return cleanup function that removes the scroll listener
- IMPORTANT: The hidden check requires `currentScrollY > 80` to prevent hiding when at the top of the page

**Body scroll lock effect** (`useEffect` with `[menuOpen]` dependency):
- When `menuOpen` is true: `document.body.style.overflow = 'hidden'`
- When false: `document.body.style.overflow = ''`
- Cleanup: reset overflow to `''`

**Close menu on route change** (`useEffect` with `[pathname]` dependency):
- Call `setMenuOpen(false)` when pathname changes
- This handles the "close on nav link click" requirement (D-10) naturally

**Desktop header JSX structure:**
```
<header> (fixed top-0 left-0 right-0 z-50, transition-all duration-300)
  <!-- Gradient overlay: only when !scrolled -->
  <div> (absolute inset-0, pointer-events-none, white-to-transparent gradient)

  <!-- Nav content wrapper: relative z-10, max-w-[1200px] mx-auto, py-6 px-6 md:px-8 -->
    <!-- Site name: Link to "/", type-title class, centered -->
    <Link href="/" className="type-title block text-center">JENNIE SLADE</Link>

    <!-- Desktop nav: hidden on mobile, flex on md+ -->
    <nav aria-label="Main navigation" className="hidden md:flex justify-center items-center gap-8 mt-3">
      {NAV_LINKS.map(link => (
        <Link
          key={link.href}
          href={link.href}
          className={`type-heading transition-opacity duration-150 ${
            pathname === link.href ? 'opacity-100' : 'opacity-60 hover:opacity-100'
          }`}
        >
          {link.label}
        </Link>
      ))}
    </nav>

    <!-- Hamburger button: visible on mobile only (md:hidden) -->
    <button
      onClick={() => setMenuOpen(true)}
      className="md:hidden absolute right-6 top-1/2 -translate-y-1/2 p-2"
      aria-label="Open menu"
      aria-expanded={menuOpen}
      aria-controls="mobile-menu"
    >
      <Menu size={24} className="text-charcoal" />
    </button>
```

**Header background classes** (computed, NOT dynamic):
- When `scrolled` is true: `bg-off-white` (solid off-white per D-04)
- When `scrolled` is false: `bg-transparent` (transparent over hero)
- When `hidden` is true: `-translate-y-full` (slide out of view)
- When `hidden` is false: `translate-y-0` (in view)
- Apply via a joined class string, not template literals for the Tailwind class names themselves

**Gradient overlay** (only visible when `!scrolled`):
- `background: linear-gradient(to bottom, rgba(255,255,255,0.7) 0%, transparent 100%)`
- Applied via inline style (gradient is not a standard Tailwind utility)
- Only render when `!scrolled` (per D-04)

**Mobile menu overlay JSX** (inline in the same component, NOT a separate file):
```
<div
  id="mobile-menu"
  role="dialog"
  aria-modal="true"
  aria-label="Navigation menu"
  className={`fixed inset-0 z-[60] bg-white flex flex-col items-center
    transition-all duration-300
    ${menuOpen ? 'opacity-100 visible' : 'opacity-0 invisible pointer-events-none'}`}
>
  <!-- X close button: absolute top-6 right-6 -->
  <button onClick={() => setMenuOpen(false)} aria-label="Close menu" className="absolute top-6 right-6 p-2">
    <X size={24} className="text-charcoal" />
  </button>

  <!-- Site name at top -->
  <Link href="/" className="type-title mt-16" onClick={() => setMenuOpen(false)}>
    JENNIE SLADE
  </Link>

  <!-- Nav links stacked vertically centered -->
  <nav className="flex flex-col items-center justify-center flex-1 gap-[48px]">
    {NAV_LINKS.map(link => (
      <Link
        key={link.href}
        href={link.href}
        className={`type-heading text-lg transition-opacity duration-150 ${
          pathname === link.href ? 'opacity-100' : 'opacity-60 hover:opacity-100'
        }`}
      >
        {link.label}
      </Link>
    ))}
  </nav>

  <!-- Instagram icon at bottom -->
  <Link
    href="https://instagram.com/jenniesladephoto"
    target="_blank"
    rel="noopener noreferrer"
    aria-label="Jennie Slade Photography on Instagram"
    className="mb-12 text-charcoal hover:text-teal-sage transition-colors duration-150"
  >
    <Instagram size={24} />
  </Link>
</div>
```

**Mobile menu z-index:** `z-[60]` so it sits ABOVE the header's `z-50`. The X button inside the overlay handles closing.

**Accessibility checklist (implement all):**
- `<nav>` element with `aria-label="Main navigation"` on the desktop nav
- Hamburger button: `aria-label="Open menu"`, `aria-expanded={menuOpen}`, `aria-controls="mobile-menu"`
- Mobile overlay: `id="mobile-menu"`, `role="dialog"`, `aria-modal="true"`, `aria-label="Navigation menu"`
- X button: `aria-label="Close menu"`
- Instagram link: `aria-label="Jennie Slade Photography on Instagram"`
- Touch targets: hamburger and X buttons have `p-2` padding for minimum 44px touch target

**IMPORTANT anti-patterns to avoid:**
- Do NOT import any fonts directly. Use `.type-title` and `.type-heading` CSS classes only.
- Do NOT use `display: none` for mobile menu. Use `opacity` + `visibility` for CSS transitions.
- Do NOT use `gray-*` Tailwind utilities. Use only brand color tokens (text-charcoal, bg-off-white, etc.) per DESN-02.
- Do NOT use Framer Motion. Pure CSS transitions only per D-05.
- Do NOT forget the scroll listener cleanup in useEffect.
- Do NOT forget `{ passive: true }` on the scroll listener.
