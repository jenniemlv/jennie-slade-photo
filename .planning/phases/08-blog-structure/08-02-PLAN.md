**A. Create `src/components/blog/BlogPostLayout.tsx`** — Server Component (no 'use client'):

Layout wrapper for individual blog posts. Receives BlogPostMeta and children (rendered MDX):

```typescript
interface BlogPostLayoutProps {
  meta: BlogPostMeta
  children: React.ReactNode
}
```

Structure:
- Section component with variant="default" (cream background)
- Narrower content width for readability: `max-w-[750px] mx-auto` inside Section's default max-w-1200
- **Header area:**
  - Date in type-heading style (small uppercase, muted gray)
  - H1 title in type-title style (this ensures proper heading hierarchy per BLOG-02)
  - Excerpt below title in type-body text-gray italic (editorial subtitle feel)
  - Spacing: mb-12 after header before featured image
- **Featured image:**
  - CloudinaryImage with src={meta.featuredImage}, width={1200}, height={800}, alt={meta.title}
  - aspect-[3/2] with object-cover, w-full
  - mb-12 after image before content
- **Content area** (children = rendered MDX):
  - Wrap in a `<div className="blog-prose">` using custom blog-prose styles (see step E below)
  - DO NOT use @tailwindcss/typography plugin
- **Back link** at bottom: "Back to The Journal" link to /blog, type-subheading style, mt-16

**B. Create `src/app/blog/[slug]/page.tsx`** — Server Component:

1. **generateStaticParams** — calls getAllPosts(), returns array of `{ slug }` objects for static generation at build time.

2. **generateMetadata** — calls getPostBySlug(params.slug), returns:
   ```typescript
   {
     title: `${post.meta.title} | Jennie Slade Photography Blog`,
     description: post.meta.excerpt,
     openGraph: {
       title: post.meta.title,
       description: post.meta.excerpt,
       type: 'article',
       publishedTime: post.meta.date,
       images: [{ url: cloudinaryUrl(post.meta.featuredImage), width: 1200, height: 630 }],
     },
   }
   ```
   For the OG image URL, construct the Cloudinary URL manually: `https://res.cloudinary.com/${process.env.NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME}/image/upload/w_1200,h_630,c_fill/${publicId}` — this gives a pre-transformed OG-sized image.

3. **Page component** — async function:
   - Call getPostBySlug(params.slug)
   - If null, call notFound() from 'next/navigation'
   - Render MDX using `MDXRemote` from 'next-mdx-remote/rsc':
     ```typescript
     import { MDXRemote } from 'next-mdx-remote/rsc'
     ```
   - Pass custom MDX components to override default HTML elements (per BLOG-04):
     - `img`: Replace with CloudinaryImage. The MDX author writes standard markdown images where the src is a Cloudinary public ID: `![Alt text](cloudinary-public-id)`. The custom img component renders CloudinaryImage with width={1200}, height={800}, the alt from markdown, and the src (public ID) from markdown.
     - `h2`, `h3`: Apply brand typography classes to ensure heading hierarchy
   - Wrap in BlogPostLayout passing meta
